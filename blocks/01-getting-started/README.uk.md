## Що таке TypeScript?

TypeScript — це JavaScript **із доданою згори системою типів** (type system). Ти пишеш
файли `.ts`, **компілятор** (compiler) TypeScript перевіряє твої типи, а потім *стирає*
їх, видаючи звичайний `.js`, який працює всюди, де працює JavaScript. Типи існують лише
на етапі розробки — вони ловлять помилки ще до того, як код колись запуститься.

```
твій код (.ts)  ──tsc──▶  звичайний JavaScript (.js)
        ▲
        └── помилки типів повідомляються тут, на етапі компіляції
```

Велика перевага: компілятор каже тобі про баги (одруківки, неправильні типи аргументів,
відсутні властивості) **поки ти друкуєш**, а не о 2-й ночі в продакшені.

## Виведення типів проти явних анотацій

Не обов'язково анотувати все. TypeScript **виводить** (infers) типи, коли може:

```ts
let count = 5;        // виведено як number
count = "hello";      // ❌ Помилка: тип 'string' не присвоюється типу 'number'
```

Анотацію (annotation) додають через `: Type`, коли виведення недостатньо — найважливіше
на параметрах функцій, які TS не може вивести самостійно:

```ts
function double(n: number): number {
  return n * 2;
}
```

Тут `n: number` — це анотація параметра, а `: number` після дужок — тип, що повертається
(return type). Тип повернення часто можна вивести, тож анотувати його не обов'язково, але
це гарна практика для публічних функцій.

## Як запускати TypeScript

- **Перевірити типи всього проєкту:** `npm run typecheck` (запускає `tsc --noEmit`).
- **Запустити один файл напряму:** `npx tsx blocks/01-getting-started/src/examples.ts`.
- **Запустити тести цього блоку:** `npx vitest blocks/01-getting-started`.

`tsx` запускає файли `.ts` без окремого кроку компіляції — ідеально для експериментів.

## Головна ідея: компілятор — твій напарник

За строгих налаштувань (strict) цього репозиторію TypeScript навмисно прискіпливий.
Сприймай кожну червону хвилясту лінію як запитання: *«А я справді знаю, якого типу це
значення тут?»* Уміння відповісти на це запитання — і є вся суть навчання.

## Вправи

Відкрий [`src/exercises.ts`](./src/exercises.ts) і реалізуй кожен `// TODO`. Потім запусти:

```bash
npx vitest blocks/01-getting-started
```

1. `greet(name)` — повернути `"Hello, <name>!"` з правильно типізованими параметром і
   поверненням.
2. `add(a, b)` — повернути суму двох чисел (типізовано).
3. `toUpperCaseSafe(text)` — повернути рядок у верхньому регістрі.
4. `isEven(n)` — повернути булеве значення, чи є `n` парним.
