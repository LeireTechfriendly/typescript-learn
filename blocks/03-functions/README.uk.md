# Блок 03 — Функції

[English](./README.md) · **Українська**

> ℹ️ Джерело істини — англійська версія ([README.md](./README.md)); український переклад може незначно відставати.

Функції — це місце, де анотації справді окупаються. Цей блок про їх ретельну типізацію:
параметри, повернення, необов'язкові та стандартні параметри, rest-параметри,
*вирази-типи функцій* (function type expressions) і те, як колбеки отримують типи задарма.

## Анотації параметрів і повернення

Параметри **завжди** потребують анотацій — TS не може їх вивести. Тип повернення зазвичай
*можна* вивести, тож анотувати його не обов'язково, але корисно для ясності:

```ts
function clamp(value: number, min: number, max: number): number {
  return Math.min(Math.max(value, min), max);
}
```

## Необов'язкові та стандартні параметри

Стандартне значення (default) робить параметр необов'язковим у місці виклику, і TS
виводить його тип зі значення за замовчуванням:

```ts
function greet(name: string, greeting = "Hello"): string {
  return `${greeting}, ${name}!`;
}

greet("Ada");        // "Hello, Ada!"
greet("Ada", "Hi");  // "Hi, Ada!"
```

Параметр також можна позначити необов'язковим через `?` (його тип стає `T | undefined`).
Необов'язкові/стандартні параметри мають іти **після** всіх обов'язкових:

```ts
function f(a: number, b?: number) {}   // ок
// function g(a?: number, b: number) {} // ❌ обов'язковий параметр після необов'язкового
```

## Rest-параметри

Збирають «решту» аргументів у типізований масив:

```ts
function sum(...nums: number[]): number {
  return nums.reduce((total, n) => total + n, 0);
}
sum();          // 0
sum(1, 2, 3);   // 6
```

## Вирази-типи функцій та колбеки

Можна описати *форму* функції як тип і використати його для анотації параметра-колбека.
Тоді параметри колбека типізуються **контекстно** (contextually typed) — їх не треба
анотувати в місці виклику:

```ts
type NumberOp = (n: number) => number;

function mapArray(items: number[], fn: NumberOp): number[] {
  return items.map(fn);
}

mapArray([1, 2, 3], (n) => n * 2); // n виведено як number — анотація не потрібна
```

## Повернення `void`

Тип повернення `void` означає «той, хто викликає, не повинен покладатися на значення, що
повертається». Зокрема, функцію, яка *таки* повертає значення, усе одно можна передати
там, де очікують `void`-колбек (саме тому `arr.forEach(x => arr.push(x))` проходить
перевірку типів):

```ts
function each(items: number[], fn: (n: number) => void): void {
  for (const item of items) fn(item);
}
```
