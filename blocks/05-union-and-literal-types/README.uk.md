# Блок 05 — Об'єднання та літеральні типи

[English](./README.md) · **Українська**

> ℹ️ Джерело істини — англійська версія ([README.md](./README.md)); український переклад може незначно відставати.

Об'єднання (union) та літеральні типи — це те, як TypeScript моделює «один із фіксованого
набору варіантів» — основа безпечних скінченних станів і даних із дискримінантом.

## Об'єднання (union)

Об'єднання `A | B` містить значення, яке є **одним із** типів:

```ts
function formatId(id: string | number): string {
  return typeof id === "number" ? id.toString() : id;
}
```

Зазвичай треба **звузити** (narrow) об'єднання (перевірити, який саме член ти маєш) перед
використанням — тут це робить `typeof`. Більше про звуження — у блоці 09.

## Літеральні типи

Літеральний тип — це *одне конкретне значення*, використане як тип:

```ts
let dir: "left" | "right";
dir = "left";   // ок
// dir = "up";  // ❌ Тип '"up"' не присвоюється типу '"left" | "right"'
```

Поєднання літералів у об'єднання моделює скінченний набір допустимих станів:

```ts
type Light = "red" | "yellow" | "green";
```

Тепер компілятор відхиляє будь-яке значення поза цим набором — одруківки стають помилками
компіляції.

## `as const`

За замовчуванням `["a", "b"]` виводиться як `string[]`. Додавання `as const` «заморожує»
його в **readonly-кортеж літералів**, з якого можна вивести об'єднання через
`typeof`/індексний доступ:

```ts
const ROLES = ["admin", "user", "guest"] as const;
type Role = (typeof ROLES)[number]; // "admin" | "user" | "guest"
```

Так зберігається єдине джерело істини: масив під час виконання й тип залишаються
синхронними.
